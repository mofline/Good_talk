adb shell am force-stop パッケージ名
adb shell am start -n パッケージ名/アクティビティ名
adb shell am force-stop com.teslacoilsw.launcher
adb shell am start -n com.teslacoilsw.launcher/.NovaLauncher

・adbで指定されたアプリを開く

novaとかを最初に開く時に使う

パッケージ名、アクティビティ名は後述のコマンドで調べる

adb shell am start -n パッケージ名/アクティビティ名

・憎き翻訳機アプリのホーム画面変化を抑える。（強制）

 無効化

一旦無効化してnovaを「常時」にしたあとに有効化すれば翻訳機を開かない限りはずっとnovaに固定される(なおふとした瞬間に勝手に起動する模様)

まぁほぼ翻訳機として使おうとなんて思う人いないだろうし戻す必要はそこまでないだろう

adb shell pm disable-user link.zhidou.translator

 L・有効化

adb shell pm enable link.zhidou.translator

・最小幅を変更する

あまりに数値が大きすぎるとちょっとあれだからやめようね

adb shell wm density 最小幅

 L・最小幅を調べる

左側が最小幅

adb shell wm size

 L・リセット

adb shell wm density reset

・デフォルトのランチャーアプリを変更する

タスクは殺してから設定する

adb shell am force-stop パッケージ名

adb shell am start -n パッケージ名/アクティビティ名

  L・手っ取り早くnovaのやつ

adb shell am force-stop com.teslacoilsw.launcher

adb shell am start -n com.teslacoilsw.launcher/.NovaLauncher

・パッケージ名を調べる

入っているアプリのパッケージ名の一覧が出てくる

それっぽいのを探そう

わかんなかったらとりあえず起動してこう

adb shell pm list packages

 L・サードパーティ製のみ

adb shell pm list packages -3

 L・システムアプリのみ

adb shell pm list packages -s

 L・無効にしたアプリのみ

adb shell pm list packages -d

 L・無効にしていないアプリのみ

adb shell pm list packages -e

・アクティビティ名を調べる

いっぱい出てくるが「android.intent.action.MAIN:」と書かれたところを探す

そこのパッケージ名のあとに書いてあるのがアクティビティ名として使える

adb shell pm dump パッケージ名

・アプリをインストールさせる

「adb install 」って打ってエクスプローラから直接ドラッグアンドドロップするだけ

adb install apkのファイルパス

・通知バーを下ろす

こいつなぜかスワイプでステータスバーを降ろせない

そんなときに役立つのがこのコマンド

強制的にステータスバーを降ろさせる(スワイプでできるようになるとは言ってない)

adb shell service call statusbar 1

・androidの詳細情報

androidのバージョンとかわからないときに使う

 L・androidバージョン確認

adb shell getprop ro.build.version.release

 L・ビルド番号確認

adb shell getprop ro.build.version.incremental

 L・コードネーム確認

adb shell getprop ro.build.version.codename

 L・SDKバージョン確認

adb shell getprop ro.build.version.sdk

 L・ビルドID確認

adb shell getprop ro.build.id

 L・センサー確認

adb shell dumpsys sensorservice
